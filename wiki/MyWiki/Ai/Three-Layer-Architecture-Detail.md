# 三层架构详解

## 定义

LLM Wiki 系统采用三层架构设计，将知识管理分为原始素材层、Wiki 知识层和 Schema 规则层，实现了关注点分离和职责清晰。

## Layer 1: Raw Sources（原始素材层）

**特性**：
- 不可变的原始文档
- 包括文章、论文、图片等多种格式
- 作为知识的源头和真实性保证
- 永久保留，不被 LLM 修改

**作用**：提供可追溯的知识来源，防止信息在摘要过程中完全丢失

## Layer 2: The Wiki（知识层）

**特性**：
- LLM 生成和维护的 Markdown 文件集合
- 包含摘要、实体页面和交叉引用
- LLM 完全负责这一层的维护
- 动态演化，持续更新

**内容类型**：
- 概念定义页面
- 实体介绍页面
- 摘要和综述页面
- 双向链接和关系网络

## Layer 3: The Schema（规则层）

**特性**：
- 配置文件，定义 Wiki 的结构和工作流程
- 包含页面格式规范、操作规则、质量标准
- 相当于系统的「宪法」
- 由人类设计和调整

**内容**：
- Frontmatter 规范
- Ingest/Query/Lint 操作规则
- 页面创建和更新标准
- 语言和格式要求

## 架构优势

1. **职责分离**：原始素材保真，Wiki 提供结构，Schema 定义规则
2. **可维护性**：每层独立演化，互不干扰
3. **可扩展性**：可以灵活调整规则而不影响已有知识
4. **可追溯性**：始终可以回溯到原始素材验证信息
