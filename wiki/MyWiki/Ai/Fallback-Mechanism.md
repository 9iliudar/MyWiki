# Fallback 机制（降级机制）

一种容错策略，通过多级备用方案确保系统在主流程失败时仍能提供基本功能。

## 定义

Fallback 机制是指当主要处理流程失败时，系统自动尝试备用方案的策略。在 [[Harness-Engineering]] 中，这是实现 [[熵管理]] 的重要手段。

## 在 LLM Wiki 中的实现

LLM Wiki 的 `json_utils.py` 实现了 3 级 fallback 解析：

### 第一级：标准 JSON 解析
- 尝试直接解析 LLM 输出为 JSON
- 适用于格式完全正确的情况

### 第二级：提取 JSON 代码块
- 从 Markdown 代码块中提取 JSON
- 处理 LLM 添加了额外说明的情况

### 第三级：修复常见错误
- 修复缺失的引号、逗号等
- 处理格式轻微偏差的情况

## 设计原则

### 逐级降级
- 优先尝试最严格的解析
- 失败后逐步放宽要求
- 确保基本功能可用

### 保持确定性
- 每一级的行为都是可预测的
- 不引入新的随机性
- 降级路径清晰明确

### 记录失败
- 记录哪一级成功
- 便于分析和优化
- 帮助改进主流程

## 价值

- **鲁棒性**：提高系统容错能力
- **可靠性**：确保 [[价值稳定输出]]
- **用户体验**：减少因格式问题导致的失败
- **熵控制**：将 AI 的随机性转化为可处理的偏差
