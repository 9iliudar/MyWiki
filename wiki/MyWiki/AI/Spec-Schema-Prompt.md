# Spec-Schema-Prompt 三层模型

## 定义

这是 [[SDD]] 方法论中的核心执行框架，通过三个层次将抽象需求转化为可执行的模型指令。金句：**Spec 管原则，Schema 管结构，Prompt 管执行**。

## 三层详解

### Spec（规格说明）

- **职责**：管原则
- **视角**：黑盒视角，关注外部行为
- **回答问题**：系统应该做什么？做到什么程度算对？
- **内容**：目标定义、边界约束、验收标准、规则说明
- **特点**：偏原则、偏整体约束、是执行的依据

### Schema（结构规则）

- **职责**：管结构
- **视角**：白盒视角，关注内部组织
- **回答问题**：数据或内容应该长什么样？结构是否合法？
- **内容**：字段定义、类型约束、嵌套关系、格式规范
- **特点**：偏结构、偏内部组织、是合法性的标准

### Prompt（执行指令）

- **职责**：管执行
- **视角**：执行层，面向具体任务
- **回答问题**：这次任务具体要怎么做？
- **内容**：将 Spec 和 Schema 翻译为模型可理解的即时操作指令
- **特点**：偏执行、偏任务级、是模型的直接输入

## 三层协作逻辑

```
Spec（定义目标和规则）
  ↓
Schema（约束结构和格式）
  ↓
Prompt（驱动模型执行）
```

- Spec 提供「做什么」的原则
- Schema 提供「长什么样」的标准
- Prompt 提供「怎么做」的指令

三者共同构成从需求到执行的完整链条，确保开发过程的可控性和可验证性。

## 实践价值

- **减少歧义**：每层职责清晰，避免混淆
- **提高复用**：Spec 和 Schema 可跨任务复用，只需调整 Prompt
- **便于验证**：每层都有明确的检查点
- **支持迭代**：可以独立优化某一层而不影响其他层
